# OptiSYS v0.2.0 — Implementation-Ready Build Plan

> **Extends** `docs/2026-05-29-optisys-overhaul-design.md` (the overhaul design). That doc settled
> *what* and *why* (§c DECISIONS RESOLVED 2026-05-29). This doc settles *how* — exact files, P/Invokes,
> AppHost lines, Settings flags, engine switch cases, snapshot/revert shape, and a CI-feasible test plan
> for each feature. **Do not re-read the design's rationale here; this is the build sheet.**
>
> **Grounding rule:** every feature below names what already exists (reuse-as-is / adapt / reference) vs
> what is genuinely new. The largest single fact: **6 battery domains are fully implemented and wired into
> the engine's `IsDomainEnabled` switch + `BuildDefaultDomains`, but are NOT registered in
> `AppHost.ConfigureServices`** — so in production DI injects only 3 domains and the 6 are dead code at
> runtime (verified: `AppHost.cs:80-82` registers exactly EcoQos/TimerResolution/MemoryOptimizer; the
> engine's DI-list-vs-fallback guard means `BuildDefaultDomains` is never called in production). Most of
> v0.2.0 is *activation + UI + tests*, not greenfield code.

# A. Self-Provisioning Task Scheduler Admin Elevation  *(FIRST — prereq for B/C/D)*

Resolves design §6 + §c-decision-2: **silent-on-logon via a HighestAvailable Task Scheduler task the app
provisions itself.** One UAC prompt at first admin-use; every logon after is elevated and silent. Graceful
unelevated degradation. This must land first because B, C, and D all need an elevated token to write HKLM
power-scheme values and SCM start-types.

### (1) REUSE MAP
- **Reuse-as-is:** `--background` flag parsing (`App.xaml.cs:88-90`, `IsBackgroundLaunch`); `StartupLog`
  (`Services/StartupLog.cs`); `PrivilegeManager.EnableAllRequired` (`OptiSYS.Core/Services/PrivilegeManager.cs`) —
  becomes a no-op safety call once Task Scheduler launches us elevated; `app.manifest` stays **`asInvoker`**
  (do NOT switch to requireAdministrator — breaks Explorer drag-drop via UIPI and forces UAC on every autostart).
- **Adapt:** `IStartupRegistrationService.Apply(bool)` shape (`StartupRegistrationService.cs:5-8`) — mirror it
  for the task backend; `AppHost` DI registration block (`AppHost.cs:99-102`); `AppRuntimeCoordinator.StartCoreAsync`
  (calls `_startup.Apply(StartWithWindows)`) — extend to choose Run-key vs task backend.
- **Reference-only:** optiBAT `TaskSchedulerHelper.cs` (schtasks `/Create /XML` with `RunLevel=HighestAvailable`
  + `LogonTrigger`; `/Run /Query /Delete`) and optiBAT `App.xaml.cs:71-116` elevation flow (IsRunningAsAdmin →
  try-run-task → runas fallback). We adopt the **dahall `TaskScheduler` NuGet (2.12.2)** instead of schtasks
  shell-out (structured idempotency, no locale-sensitive text parsing).
- **Build new:** `Services/Elevation/TaskSchedulerService.cs`, `ITaskSchedulerService`, `ElevationHelper`,
  the `--provision-elevation` startup branch, `Settings.UseTaskScheduler`, installer uninstall hook.

### (2) IMPLEMENTATION

**NuGet:** add `TaskScheduler` 2.12.2 to `OptiSYS.App.csproj` (net9.0-windows served via net6.0-windows TFM).

**New `src/OptiSYS.App/Services/Elevation/ITaskSchedulerService.cs`**
```csharp
public interface ITaskSchedulerService
{
    bool IsElevated { get; }                 // WindowsPrincipal.IsInRole(Administrator)
    bool TaskExists();                       // FindTask("OptiSYS") != null AND action path == ProcessPath
    bool NeedsProvisioning();                // !TaskExists() || RunLevel != Highest || path mismatch
    void CreateOrUpdateTask();               // MUST be called from an elevated process
    void DeleteTask();                       // uninstall / opt-out
}
```

**New `TaskSchedulerService.cs`** — task name `"OptiSYS"`, root folder, idempotent via `TaskCreation.CreateOrUpdate`:
```csharp
using var ts = new TaskService();
var td = ts.NewTask();
td.Principal.UserId    = WindowsIdentity.GetCurrent().Name;
td.Principal.LogonType = TaskLogonType.InteractiveToken;   // no stored password, interactive session (tray visible)
td.Principal.RunLevel  = TaskRunLevel.Highest;
td.Triggers.Add(new LogonTrigger { UserId = WindowsIdentity.GetCurrent().Name, Delay = TimeSpan.FromSeconds(7) });
td.Actions.Add(new ExecAction(Environment.ProcessPath, "--background", null));
td.Settings.ExecutionTimeLimit       = TimeSpan.Zero;       // tray app, never auto-kill
td.Settings.DisallowStartIfOnBatteries = false;
td.Settings.StopIfGoingOnBatteries     = false;
ts.RootFolder.RegisterTaskDefinition("OptiSYS", td, TaskCreation.CreateOrUpdate, null, null, TaskLogonType.InteractiveToken);
```
- **Existence check** uses `FindTask("OptiSYS")` (returns null, never throws) — NOT `GetTask` (throws
  `FileNotFoundException`, dahall issue #79). Wrap in try/catch → conservative "needs provisioning" on any COM error.
- **Path staleness:** `NeedsProvisioning()` compares the task's `ExecAction.Path` to `Environment.ProcessPath`;
  mismatch (in-place upgrade moved the exe) → re-register.

**New `Services/Elevation/ElevationHelper.cs`** — the runas bridge:
```csharp
public static void RequestProvisioning()
{
    var psi = new ProcessStartInfo {
        FileName = Environment.ProcessPath, Arguments = "--provision-elevation",
        UseShellExecute = true, Verb = "runas"
    };
    try { Process.Start(psi)?.WaitForExit(); }
    catch (Win32Exception ex) when (ex.NativeErrorCode == 1223) { /* user clicked No → caller shows InfoBar */ }
}
```

**Edit `App.xaml.cs` OnLaunched (before `AppHost.Initialize()`):** early-detect the provisioning arg:
```csharp
var cmd = Environment.GetCommandLineArgs();
if (cmd.Any(a => string.Equals(a, "--provision-elevation", StringComparison.OrdinalIgnoreCase))) {
    new TaskSchedulerService().CreateOrUpdateTask();   // we ARE elevated here (runas launched us)
    Environment.Exit(0);                               // never opens a window
}
```
Also compute `IsElevated` once and store it in a DI singleton for domains to gate on.

**Edit `AppRuntimeCoordinator.StartCoreAsync`** — backend selection:
- `Settings.UseTaskScheduler == true`:
  - if `NeedsProvisioning()` && `IsElevated` → `CreateOrUpdateTask()` inline (already elevated → no UAC; this is
    the GP/EDR-deletion self-heal path).
  - if `NeedsProvisioning()` && `!IsElevated` → do NOT prompt at boot; set `Settings.ElevationPending=true` so the
    UI shows a non-nagging "Grant admin access" InfoBar; user click → `ElevationHelper.RequestProvisioning()`.
  - also call `_startup.Apply(false)` to **remove the HKCU Run key** (avoid double-launch).
- `Settings.UseTaskScheduler == false`: current Run-key path (`_startup.Apply(StartWithWindows)`); `DeleteTask()` if present.

**Settings flags** (`Settings.cs`, near `StartWithWindows` line 148):
```csharp
public bool UseTaskScheduler { get; set; } = false;   // opt-in; user enables "Run with highest privileges"
public bool ElevationPending  { get; set; } = false;  // UI banner state, not persisted intent
```

**AppHost registration** (`AppHost.cs`, after line 102):
```csharp
sc.AddSingleton<ITaskSchedulerService, TaskSchedulerService>();
```

**Installer uninstall** (`OptiSYS.Installer/MainWindow.xaml.cs` RunUninstallAsync ~line 308, beside the Run-key delete):
add `schtasks /Delete /TN "OptiSYS" /F` (or COM `TaskService.RootFolder.DeleteTask("OptiSYS", false)`). The
installer is `asInvoker` and CANNOT create a HighestAvailable task — **task creation lives only in the app's
`--provision-elevation` path**, never in the installer.

**Graceful degradation contract:** every admin-needing domain (B/C/D) reads `ITaskSchedulerService.IsElevated`
in `IsSupported`/`Apply`; when false → skip silently + report "needs admin" in `GetStatus().Summary`. The
safe-by-default set (GPU-power, memory, PowerBaseline overlay/EcoQoS) stays fully functional unelevated.

# B. Battery-Baseline Domain (`power-baseline`) + `PowerWriteACValueIndex`

New safe, AC/DC-aware, fully reversible power-plan domain. Ships **ON for battery devices** (per design §4 /
§b). All writes are admin-gated (HKLM power scheme) → depends on **A**.

### (1) REUSE MAP
- **Reuse-as-is:** `CpuParkingDomain` / `DiskIoCoalescingDomain` per-setting snapshot pattern (read DC index →
  `DomainSnapshot.Set` → write → `PowerSetActiveScheme` → `Revert` writes captured back); `NativeMethods` helpers
  `GetActiveScheme()` (`:563`), `ReadDCValue()` (`:577`), `WriteDCValue()` (`:583`); `IPowerSourceMonitor`
  (`AppHost.cs:88-89`) for AC↔DC; the `0=never` sentinel handling for timeouts.
- **Adapt:** the `WriteDCValue` helper into a sibling `WriteACValue` / `ReadACValue`; `DomainSnapshot` typed bag.
- **Reference-only:** optiBAT power-setting GUIDs (`optibat/.../NativeMethods.cs:68-88`) — but OptiSYS already has
  the processor/disk GUIDs; we add only the new subgroups/settings below.
- **Build new:** `Domains/Battery/PowerBaselineDomain.cs`; `PowerWriteACValueIndex` + `PowerSetActiveOverlayScheme`
  + `PowerGetEffectiveOverlayScheme` P/Invokes + GUID constants for PERFAUTONOMOUS, PERFEPP, PCIe ASPM, USB-suspend
  power-plan, VIDEOIDLE, STANDBYIDLE; overlay GUIDs.

from `powercfg /query`.

**AppHost:** `sc.AddSingleton<IOptimizationDomain, PowerBaselineDomain>();` (registration order: after
GPU/CPU/disk so it is the single overlay owner; coordinate so it never re-writes PROCTHROTTLEMIN).

## D1. WiFi Optimizer (`wifi-optimizer`) — unelevated, session-scoped

### (1) REUSE MAP
- **Reuse-as-is:** `IOptimizationDomain` lifecycle; `DomainSnapshot` per-interface boolean capture;
  `IPowerSourceMonitor`-style callback marshalling discipline (DispatcherQueue).
- **Reference-only:** `NetworkPowerDomain` (registry-based, NOT wlanapi — confirms WLAN is greenfield).
- **Build new (everything):** `Native/WlanNativeMethods.cs` (entire wlanapi surface — zero exists in tree);

### (2) IMPLEMENTATION
**New `Native/WlanNativeMethods.cs`** — `WlanOpenHandle(dwClientVersion=2)`, `WlanCloseHandle`,
`WlanEnumInterfaces`, `WlanSetInterface`, `WlanQueryInterface`, `WlanFreeMemory`, `WlanRegisterNotification`;
structs `WLAN_INTERFACE_INFO[_LIST]`, `WLAN_NOTIFICATION_DATA`; enums `WLAN_INTF_OPCODE`
(`background_scan_enabled=0x2`, `media_streaming_mode=0x3` — confirmed ordinals), `WLAN_INTERFACE_STATE`
(`connected=1`), `WLAN_NOTIFICATION_ACM` (`connection_complete=10`). pData for both opcodes = `BOOL` (4-byte int).
**Critical rules baked in:** use **`WLAN_NOTIFICATION_SOURCE_ACM` (0x8) only — never MSM (0x10)** (24H2 location
gating → ERROR_ACCESS_DENIED); keep a `GCHandle` on the callback delegate (collection → access violation);
free every `ppData` with `WlanFreeMemory` (never `Marshal.FreeHGlobal`); **never** call `WlanRegisterNotification`
from inside the callback (deadlock).

**New `Domains/Network/WiFiOptimizerDomain.cs`** — `Id="wifi-optimizer"`, `Category="Network"`,
`IsSupported = Wlansvc running + ≥1 WLAN interface`. Apply: enum interfaces → per connected interface
(`state==connected`, else skip → ERROR_INVALID_STATE) capture both booleans (`WlanQueryInterface`) → set
background_scan=FALSE, media_streaming=TRUE → **verify-after-write** (`WlanQueryInterface` again, compare;
mismatch = Intel/MediaTek silent-ignore → log + `GetStatus` note "driver does not support background-scan control",
do NOT fail). Register ACM notification; on `connection_complete` re-apply on the new interface GUID. Revert:
restore captured booleans then `WlanCloseHandle` (handle close auto-reverts session opcodes — crash-safe). Settings:
`WiFiOptimizerEnabled=false`, `WiFiDisableBackgroundScan=true`, `WiFiStreamingMode=true`. Switch:
`"wifi-optimizer" => _settings.WiFiOptimizerEnabled,`. AppHost: register singleton. UI: folded under
"Power & Energy" (design §c-6); single "Undo all WiFi tweaks"; show live queried state.

### (3) TEST PLAN

### (2) IMPLEMENTATION (by verdict)

**SAFE-BY-DEFAULT — ship ON:**

  (alpha≈0.2) + slope over `TrendWindowSize` (exists, =10) + `CommitTotal/CommitLimit > 0.65` → fire a *Conservative*

**OPT-IN — ship OFF:**
- **5-level memory priority + foreground restore** (Rec.2): VERY_LOW(1) for a tighter pure-daemon sub-allowlist;

**REJECT (do not implement):** PageCombining automatic-path trigger (keep deepClean-only gate); MMAgent

**Battery algorithm improvements (reuse opportunities from the engine map):**
